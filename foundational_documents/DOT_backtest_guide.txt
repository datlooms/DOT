 3. THE ANTI-CURVE-FIT SELECTION (the method — reproduce it exactly)
═══════════════════════════════════════════════════════════════════════
  This is how BOOK-50 was selected from the gated field. Do not substitute a
  different procedure; the discipline IS the product.

  3.1  PERSISTENCE AT THREE SCALES. Require, simultaneously: 6/6 monthly folds
       positive, high ISO-week positivity, and all-weekday coverage. The ≥30-trade
       floor bars small-sample flukes. A signal that clears only one scale is not
       persistent.

  3.2  LOSS-DECORRELATION SELECTION (the core method — NOT return correlation,
       NOT in-sample PF). Build per-day loss vectors. Greedily add the signal
       whose addition keeps the book's worst-day highest — i.e. whose losing days
       are most INDEPENDENT of the book so far. This rewards independence, not
       sample-fit, and cannot be gamed by memorizing the sample.
       SANITY: a naive union of all persistent signals collapses to PF ~1.9;
       the loss-decorrelated book holds above 5. The edge lives in the
       independence, not the count.

  3.3  OUT-OF-SAMPLE GATE. Select on Jan–Apr; measure on unseen May–Jun. A
       curve-fit book collapses OOS; a real one holds or strengthens. The core
       books strengthen OOS (diamond 5.54; BOOK-40 6.54; BOOK-48 6.65). Treat OOS
       as one input, not the sole gate — but a book that fails OOS is rejected.

  3.4  LEAVE-ONE-OUT TRIM (fight overfitting actively). For each signal in a
       widened book, remove it and re-measure the BOOK (not standalone). Cut the
       signals whose removal RAISES book PF/OOS-PF — the loss-correlated dead
       weight — even when their standalone PF is high. PROTECT disproportionate
       bear-fold contributors. This is how BOOK-58 -> BOOK-48 recovered quality.

  3.5  NULL-HONORING. Test against circular-shift / permutation nulls and honor
       the result. Depth failed its null (pure-depth PF ~1.1; depth-without-
       triples PF < 1) — excluded. The flip-trajectory model scored AUC 0.492
       (below chance) — killed. Report your own nulls; a process that hides them
       is manufacturing signal.

  3.6  STRUCTURE COVERAGE (last, and only if it doesn't compromise the above).
       Classify signals into the 8 DOTS structures. If a structure is missing,
       fill it ONLY with a signal that clears persistence + clean decorrelation +
       holds the book worst-day. The 2 F1 fills (SQUEEZE_BREAKOUT, TREND_
       EXHAUSTION) each hold 6/6, high-weekly, streak ≤ 1, and the −$127.5
       worst-day. Do not force a structure with a signal that stacks losses.

  3.7  NULL / SHUFFLE TEST (run on the FINISHED book — the luck / curve-fit detector).
       Break the signal->outcome link three ways, keeping the EXACT trade management
       and trade count, 200+ iterations each, to build a null distribution:
         (i)   direction-shuffle — real entry bars, randomize each trade's long/short.
         (ii)  random-timing — enter on random eligible bars (same D2D gate, same
               count) instead of real signal bars. This is the TM-ALONE test.
         (iii) fully random — random bars AND random direction.
       BEYOND-NULL-MAX LOGIC: a real edge sits beyond the MAXIMUM of every null (not
       one random arrangement out of hundreds matches it); a fit / lucky book sits
       INSIDE the null cloud. SIGNAL-vs-TM ATTRIBUTION: if random-timing through the
       real TM already makes the money, the TM is the edge (bad); if random entries
       lose or break even and only the real signals win, the SIGNAL is the edge (good).
       BOOK-50: beyond the max of all three nulls (0/200), ~16σ above the
       direction-shuffle mean and ~35σ above the random means, p < 0.005; TM-alone
       LOST money (net −$678, PF 0.97) — the signal is the edge, not the TM. Treat any
       book that does not clear this as a fit until proven otherwise.

       worst-day gate is evaluated before any profitability metric.

  4.7  LIVE-RISK JAR (concurrency — model it EXACTLY as the EA runs it). Admission is
       governed by a 6-lot live-risk jar, NOT a position-count cap. In the sim, track a
       running live-lot counter: +1 when a position opens; −1 at the exact bar it reaches
       break-even (SL moved to entry±step — the winner's lot leaves the jar); −1 on a
       still-live exit (a post-BE close was already freed, no double); admit a new signal
       ONLY when live lots < 6. This MUST match the EA bit-for-bit — a JAR-PARITY
       requirement, the same discipline as export=live is for thresholds: if the sim's
       admission diverges from the EA's live-lot logic, the backtest is invalid. Expect
       2,409 tr under RUN (+74 vs the old count cap) at a hard bound of 6 live lots.

  4.8  EXECUTION-MODEL PARITY (per-bar SL management — the third parity leg). The sim
       repositions the SL only at bar boundaries, off the CLOSED bar's High/Low. The live EA
       must do the SAME: reposition the SL ONLY on a new bar, off the closed bar's High/Low
       (NOT the live Bid tick, NOT the Close). The broker still holds a hard SL that executes
       intrabar on any tick (downside protection is always live per-tick); only the EA's
       REPOSITIONING is per-bar. Per-bar sequence per open position each new bar: (1) exit-check
       vs current SL, (2) tiers off closed-bar high/low, (3) arm/lock BE, (4) trail. LIVE
       PER-BAR MANAGEMENT == THE PER-BAR SIM (the $58,685 RUN book). PER-TICK management diverges
       DOWNWARD into an unvalidated bracket: with LOCK_FRAC=1.0 the lock sits on the tier-1/

       (thresholds) and jar-parity (admission): if the EA repositions per-tick, live != the
       validated book. Only tick-level data / a demo can locate live inside that bracket.

 5. TIC-PROOF STANDARDS (do not skip)
═══════════════════════════════════════════════════════════════════════
  - Never conclude on first findings. Cross-validate every key result across the
    OOS plateau, all folds, and ±20% perturbation before accepting it.
  - Believe the engine, not a proxy or a correlation. engine.run_portfolio is the
    sole ratified trade path; no alternate simulation pipelines.
  - Every cited number self-computed from source; verify byte-identity where it
    matters; do not trust upstream claims.
  - A weak or negative result is a map, not a verdict. It redirects a line of
    inquiry; it does not close the project. Re-derive from the data; do not
    inherit a prior instance's framing.
  - Reproduce, don't reinvent. If a run would exceed ~30 searches/steps of scope,
    checkpoint (atomic os.replace + .done marker) and resume; use spawn-forced
    multiprocessing for parallel scoring.

  A reproduced result that clears persistence-at-three-scales, loss-decorrelation,
  OOS, leave-one-out, and null-honoring is what "validated" means here. BOOK-50
  clears all five. The live FTMO demo is the final test the backtest cannot run.
